# NOTES — 教学偏好与备忘

## 学员画像（LR-0002 修正）
- **背景**：蓝牙**驱动移植 + 协议栈适配**工程师（host/OS 层，**不是 controller 固件层**）。了解 BT 协议但不写 PHY/MAC 固件。同时开发 **sub-GHz (sub-1 GHz) 协议栈**（窄带 FSK/LoRa 类，长距离低速率）。
- **目标**：转型 WiFi 驱动 / 协议栈方向（mac80211 / cfg80211 / nl80211 对口）
- **直接对口（可加速）**：驱动框架、协议状态机、帧解析、关联/扫描流程 —— BT/sub-GHz 经验镜像迁移
- **需补**：802.11 协议域知识（帧类型、BSS/IBSS/ESS）、OFDM 宽带 PHY（sub-GHz 是窄带，OFDM 是新路线）、CSMA/CA 退避机制（sub-GHz 多为调度/ALOHA，分布式竞争是新概念）
- **有底子的**：协议分层、状态机设计、驱动框架。⚠️ **RF 数学基础被高估**（LR-0005）：协议栈 ≠ RF/PHY，dB/链路预算/噪声计算需从零教，不能用 sub-GHz 经验假设。

## 教学偏好
- **语言**：中英混合。中文讲解为主，专业术语保留英文原文。
  - **⚠️ 翻译规范（LR-0011）**：每个专业英文术语在**每节课首次出现**时，内联写成 `English Term (中文翻译)`，例如 `CSMA/CA (载波侦听多址接入)`。中文翻译**严格对齐 GLOSSARY.md**，保证跨课一致。只在首次出现时加翻译，之后用英文（避免冗长）。纯物理单位（μs、MHz、dBm、dB、mW、km 等）不翻译。
- **节奏**：每天投入较多时间（约 1 小时/日），稳定每日课程节奏
- **考证**：不考 CWNA，纯学知识。手上的书是 CWNA-108 版（已停考，现行 CWNA-109），仍作骨架
- **⚠️ 不读原书**：学员明确表示**不希望读 CWNA 原书**，所有知识由课程网页直接交付。CWNA 书降级为「我的内部参考资料」（我读它来组织课程，但不再让学员去读）。课程末尾的「推荐资料」不要指向书章节。
- **桥梁策略**：用 BT 背景做知识桥梁（FHSS→OFDM、connection event→CSMA/CA、GFSK→OFDM 子载波、piconet→BSS），不当零基础教
- **章节排序**：固件优先级前置 —— RF 基础 → RF 数学 → 802.11 标准族 → MAC → Medium Access → PHY → 拓扑 → 其余作背景

## 课程设计规则
- 每节课短、可快完成（10–20 分钟内）
- 每节课给一个能摸到的具体收获（concept anchor）
- 每节课标注「对固件重要」的部分（区别于管理员视角）
- 检索练习题：答案长度统一，避免格式泄题
- 复用 `./assets/` 组件，不要内联重复代码

排序原则：**钉子优先 + 地基先稳**。OFDM（钉子 1）已立，RF 基础是所有 RF/PHY 课的地基，必须扎实再推进。CSMA/CA（钉子 2）是协议域，学员强项，可在 RF 稳后推进。

**教学警示（LR-0005）**：不要假设学员有 RF 数学基础。协议栈 ≠ RF/PHY。dB/dBm 每次首次出现都从定义讲起。数学容忍度需慢节奏。

**教学警示（LR-0016，新增）**：教学方对 RF 硬件层（Noise Figure、灵敏度推导）和法规（FCC/ETSI 具体数值）的记忆不可靠——与 LR-0008（LoRa SF）、LR-0005（RF 高估）同类警示。涉及具体数值（NF、法规 dBm 上限、SIFS 分量绝对值）必须查一手标准/法规。**RF 数学路线二选一**：CWNA 经验路线（实测噪声底 + 厂商灵敏度表，NF 隐含）或 RF 工程严格路线（kTB + NF + SNR_req）；当前第 4 课是两头不靠的混合体，待修订。

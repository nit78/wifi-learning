Wi-Fi 术语速查表

每个 Wi-Fi 缩写的基本理解 —— 不只是定义，而是"为什么是这个值、关键约束是什么"。 和 GLOSSARY.md（正式定义）互补：这里偏"工程直觉"，那里偏"严格定义"。

介质访问 / 时序参数（MAC）

CSMA/CA Carrier Sense Multiple Access with Collision Avoidance · 载波侦听多址接入 / 碰撞避免

Wi-Fi 的核心介质访问范式：先听（CCA）→ 等空闲 DIFS → 随机退避 backoff → 发。分布式、无中央调度，与蓝牙 TDMA 形成根本对照。

不是"完全不碰撞"，而是"尽量降低碰撞概率"（靠随机退避）。碰撞仍可能因退避撞车或隐藏节点发生。

DCF Distributed Coordination Function · 分布式协调函数

802.11 的基础 MAC 模式，基于 CSMA/CA。所有 STA 平等竞争介质，无中央调度。Wi-Fi 最常用的接入方式。QoS 版本叫 EDCA。

SIFS Short Interframe Space · 短帧间间隔

最短 IFS，最高优先级。用于进行中的一次帧交换内部（DATA→ACK、RTS→CTS）。

802.11a: 16 μs 802.11b/g: 10 μs

SIFS 由四个物理分量决定（标准公式）：

aSIFSTime = aRxRFDelay + aRxPLCPDelay + aMACProcessingDelay + aRxTxTurnaroundTime

802.11a 实测拆解：PLCP 解码是大头（12.5 μs，78%），收发切换只占 2 μs（12%）。

⚠️ 常见误解："SIFS 就是收发天线切换时间"——不准确。天线切换只是四分量之一，且非主要。真正的大头是 PHY 解码 preamble（PLCP）。

SIFS 矫正：ACC（累积纠正） 不适用于此；SIFS 是物理下限，设太短硬件来不及响应 ACK，设太长浪费信道。

DIFS DCF Interframe Space · DCF 帧间间隔

普通数据帧发起接入前要等的间隔。是"准入门槛"——必须连续空闲 DIFS，才有资格进入 backoff。

DIFS = SIFS + 2 × SlotTime

802.11a: 34 μs（16 + 2×9） 802.11b: 50 μs（10 + 2×20）

比 SIFS 长 → 新对话（用 DIFS）优先级低于进行中的对话（用 SIFS），所以 ACK 一定先抢到信道。

SlotTime Slot Time · 时隙

backoff 计数器递减的时间单位。多 STA 的退避在时隙边界对齐同步。

802.11a: 9 μs 802.11b: 20 μs

SlotTime > 传播时延 + CCA 判定时间，保证一个时隙内能侦听到别人是否开始发。

PIFS PCF Interframe Space · PCF 帧间间隔

AP 轮询用（PCF 模式，实际很少用）。PIFS = SIFS + 1×SlotTime，介于 SIFS 和 DIFS 之间。

802.11a: 25 μs

EIFS Extended Interframe Space · 扩展帧间间隔

收到错误帧（CRC 失败，看不懂）后用。比 DIFS 更长，确保不会踩到一个自己解不出的进行中交换。EIFS = SIFS + ACK 发送时间 + DIFS。

AIFS Arbitration Interframe Space · 仲裁帧间间隔

QoS（802.11e/EDCA）用，可变。不同接入类别（AC：语音/视频/尽力而为/背景）用不同 AIFS。AIFS[AC] = SIFS + AIFSN[AC] × SlotTime，AIFSN ≥ 2。语音 AIFS 最短（优先级最高）。

CW Contention Window · 竞争窗口

backoff 计数器的取值范围 [0, CW]。随机选一个数，按 slot 递减。

CWmin = 15 CWmax = 1023（802.11a）

碰撞后 CW 倍增（BEB），成功后重置回 CWmin。见下条。

BEB Binary Exponential Backoff · 二进制指数退避

每次碰撞后 CW 翻倍：CW = 2ⁿ · (CWmin + 1) − 1，n = 重试次数，封顶 CWmax。

序列：15 → 31 → 63 → 127 → 255 → 511 → 1023（封顶）

精妙之处：碰撞多时自动拉长退避（信道拥挤 → 退让），空闲时快速发送。无中心的自适应拥塞控制。

CCA Clear Channel Assessment · 信道空闲评估

PHY 层判断信道忙/闲的功能，是 CSMA/CA "先听"的物理基础。CCA 阈值决定多弱的信号算"忙"。

CCA 分两种：能量检测（ED，信号强度超阈值判忙）+ 载波侦听（CS，解到 802.11 preamble 判忙）。

RTS/CTS Request To Send / Clear To Send · 请求发送 / 允许发送

隐藏节点的缓解机制。发送方先发短 RTS，AP 回 CTS 广播"我要收数据，他人别发"，周围 STA 听到 CTS 后退避。

代价：每次发送多了两个控制帧。通常只对大数据帧启用（有 RTS threshold 阈值，默认常关闭）。

ACK Acknowledgment · 确认帧

接收方正确收到数据帧后回的确认。用 SIFS（最高优先级）发送。没收到 ACK → 发送方判定碰撞 → BEB 重试。

网络结构（Topology）

BSS Basic Service Set · 基本服务集

Wi-Fi 的基本网络单元：1 个 AP + N 个关联的 STA。结构上对应蓝牙 piconet，但控制范式相反（AP 不调度，STA 竞争）。

ESS Extended Service Set · 扩展服务集

多个 BSS 通过分布式系统（DS，通常是有线网）互连，支持漫游。同一个 SSID 下的多 AP 网络。

IBSS Independent BSS · 独立基本服务集

无 AP 的对等 Wi-Fi 网络，俗称 ad-hoc。Wi-Fi Direct 类拓扑的基础。

STA Station · 站点

任何具备 802.11 MAC/PHY 能力的设备。手机、笔记本、IoT 节点都是 STA；AP 本质上也是一种特殊 STA。

AP Access Point · 接入点

提供 STA 关联、桥接有线网络的 Wi-Fi 设备。BSS 的中心节点。注意：AP 不调度 STA 的发送时刻（不像蓝牙 master）。

BSSID Basic Service Set Identifier · 基本服务集标识

标识一个 BSS 的 6 字节 MAC。infra BSS 里 = AP 的 MAC；IBSS 里 = 随机生成。不是 SSID（SSID 是网络名字，字符串）。

MAC 帧结构（Frame Structure）

MAC Frame MAC 帧

802.11 数据链路层 PDU。固定头 24B（+可选 Addr4 6B）+ Body 0–2312B + FCS 4B。分三类：管理/控制/数据。

布局：FC(2) | Duration(2) | Addr1(6) | Addr2(6) | Addr3(6) | SeqCtrl(2) | [Addr4(6)] | Body | FCS(4)

Frame Control 帧控制字段

MAC 头前 2 字节。含 Type/Subtype（帧类型）、ToDS/FromDS、Retry、Power Mgmt、Protected Frame 等。解析帧的第一步。

Type 2bit：00=管理、01=控制、10=数据。Subtype 4bit 细分（Beacon=0080、ACK=0x0D 等）。

三类帧

管理帧 (Management)：Beacon、Probe、Authentication、Association。建立/维护连接，是状态机载体。ToDS=FromDS=0，用 3 地址。

控制帧 (Control)：ACK、RTS、CTS、PS-Poll、Block ACK。辅助数据传输，不承载数据。ACK 仅 14B。

数据帧 (Data)：承载上层。QoS Data 多 2B QoS Ctrl（含 TID，对应 EDCA AC）。

Addr1–4 四个地址字段

4 个 6B MAC 地址。5 个角色：DA(目的)、SA(源)、BSSID(BSS标识)、RA(本跳接收)、TA(本跳发送)。

铁律：Addr1=RA，Addr2=TA，恒成立。Addr3/4 由 ToDS/FromDS 决定。

为什么 4 个？AP 是桥接点，"本跳收发(RA/TA)"和"最终去来(DA/SA)"是两套地址，要分开带。以太网只有 DA/SA，因为端到端直连。

ToDS / FromDS To DS / From DS 位

Frame Control 里 2 bit，决定地址映射（背下来）：

00 管理帧：A1=DA, A2=SA, A3=BSSID

01 AP→STA 下行：A1=DA, A2=BSSID, A3=SA

10 STA→AP 上行：A1=BSSID, A2=SA, A3=DA

11 WDS 桥接：A1=RA, A2=TA, A3=DA, A4=SA（用满4个）

NAV Network Allocation Vector · 网络分配矢量

虚拟载波侦听。读帧里 Duration 字段（μs），设本地计时器。NAV 没归零就不发，即使 CCA 说闲。

STA 能不能发：物理 CCA 和 NAV 同时检查，两个都闲才行。NAV 价值：不用真听到数据帧也能知道信道被占（隐藏节点靠 CTS 的 Duration 设 NAV）。

Duration/ID 持续期 / ID

2 字节。大多数帧存 NAV 值（μs），告诉别人"这组交换还要占多久"。PS-Poll 帧里存 AID（Association ID）。

Seq Ctrl Sequence Control · 序列控制

2 字节：Fragment Number(4b) + Sequence Number(12b)。用于分片重组和去重。配合 Retry bit：同 Seq+Retry=1 是重传，接收方去重。

FCS Frame Check Sequence · 帧校验序列

4 字节 CRC-32，帧尾。校验失败丢弃帧，且对错误帧用 EIFS（而非 DIFS）退避——避免踩到看不懂的交换。

关联流程与状态机（Association Flow）

802.11 状态机三态 · State 1 / 2 / 3

STA 的官方三态：State 1 未认证 → State 2 已认证未关联 → State 3 已关联（可发数据）。每态允许发的帧分 Class 1/2/3。

State 1 Class 1：Probe/Beacon/Auth

State 2 +Class 2：Association Req/Resp

State 3 +Class 3：数据帧、Deauth/Disassoc

⚠️ "已关联" ≠ "能传数据"。WPA 网络还要过 4-Way Handshake 进入"已授权"才能真正上行。

Beacon 信标帧

AP 周期广播的管理帧。典型周期 102.4 ms。承载 SSID、BSSID、能力、速率、TIM（省电通知）。

被动扫描靠它；TIM 位图按 AID 标记哪个 STA 有缓存下行帧。

Probe Req/Resp 探测请求 / 探测响应

主动扫描的一对管理帧。STA 发 Probe Req（带想找的 SSID），匹配 AP 回 Probe Resp（内容≈Beacon）。比被动扫描更快发现网络。

SSID Service Set Identifier · 服务集标识

Wi-Fi 的"网络名"，最长 32 字节字符串。不是 BSSID（那是 MAC）。一个 ESS 下所有 AP 共用同一 SSID。

Capability Info 能力信息字段

2 字节位域，出现在 Beacon/Probe Resp/(Re)Assoc Req-Resp 里。bit 含义：ESS、IBSS、Privacy（是否加密）、Short Preamble、Short Slot Time 等。关联时 STA 与 AP 协商能力的载体。

Authentication 802.11 认证帧

关联之前的一对管理帧（不是安全认证！）。Open System（algorithm=0）两帧握手：

Seq 1 STA→AP 请求 Seq 2 AP→STA 响应（基本都成功）

⚠️ 常见误区：把它当"安全认证"。它只是状态机过场，真正身份认证由后续 WPA 4-Way Handshake / 802.1X 完成。

Association 关联

STA 加入 BSS 的一对管理帧。STA 发 Association Req（能力/速率/监听信道），AP 回 Association Resp（Status Code + AID）。

状态机视角：State 2 → State 3 的迁移。成功后 STA 才能发数据帧。

AID Association ID · 关联标识符

AP 在 Association Response 里分配的 16-bit ID。范围 1–2007。

用途：PS-Poll 帧的 Duration/ID 字段存 AID 取缓存帧；TIM 位图按 AID 标记哪个 STA 有下行缓存。

Reassociation 重关联

已关联 STA 在同一 ESS 内从原 AP 切到新 AP（漫游）的一对帧。Reassoc Req 比 Assoc Req 多一个"原 AP 的 MAC"字段。漫游（roaming）的核心帧。

Deauth / Disassoc 取消认证 / 取消关联

断连帧。Deauth：任意状态 → State 1（彻底打回）。Disassoc：State 3 → State 2（保留认证）。主动踢人/下线/超时都用它们，带 Reason Code。

省电机制（Power Save）

Power Save 省电模式

STA 在 Frame Control 置 Power Management bit=1 告诉 AP"我要睡了"，AP 为它缓存下行帧。STA 周期醒、查 TIM、发 PS-Poll 取帧。

核心思路：把"被动接收"变成"主动预约取"，睡眠期间关闭射频。

TIM Traffic Indication Map · 业务指示图

Beacon 里的位图。每一位对应一个 AID——位=1 表示 AP 为该 AID 缓存了单播帧。省电 STA 醒来解析 TIM，看自己的位。

DTIM Delivery TIM · 投递业务指示图

特殊的 TIM，按 DTIM Period（如每 3 个 Beacon 一次）出现。DTIM 不仅指示单播缓存，还立即发送缓存的广播/组播。省电 STA 必须在 DTIM 醒来，否则错过广播（如 ARP）。

PS-Poll Power-Save Poll · 省电轮询帧

省电 STA 向 AP"取一个缓存帧"的控制帧。Duration/ID 字段放 AID（不是 NAV！）。AP 收到后回一个缓存帧。多个帧就发多个 PS-Poll。

Listen Interval 监听间隔

STA 在 Association Request 里声明"我每 N 个 Beacon 醒一次"。N=1 每次都醒；N=10 睡更久但 AP 要缓存更多。AP 据此分配缓存缓冲区。

More Data 更多数据位

Frame Control 里 1 bit。AP 发缓存帧时若 STA 还有更多待取，置 More Data=1。STA 据此决定"继续醒着取完"还是"回去睡"。

U-APSD Unscheduled Automatic Power Save Delivery · 非调度自动省电投递

WMM-PS（802.11e）增强。STA 不用每次发 PS-Poll，而是用上行数据/触发帧让 AP"送货上门"。对 VoIP 双向周期业务省电显著。一次唤醒收发双向。

TWT Target Wake Time · 目标唤醒时间

802.11ax (Wi-Fi 6) 省电，源自 802.11ah。STA 和 AP 显式协商"唤醒时刻表"（Service Period），只在约定窗口收发，脱离 Beacon/DTIM 周期束缚。

可睡几十秒到几分钟，是 Wi-Fi 6 IoT（智能门锁、传感器）续航关键。

安全 / 加密（Security）

WPA / WPA2 / WPA3 Wi-Fi Protected Access

WFA 安全认证品牌。WPA1（2003，TKIP 过渡）WPA2（2004，802.11i，CCMP，最普及）WPA3（2018，SAE + 强制 PMF）。

WPA 不是 IEEE 标准号，是 WFA 认证品牌；背后对应 802.11i 等修正案。

CCMP Counter Mode CBC-MAC Protocol · 基于 AES 的加密+完整性

WPA2 强制加密协议。同时做加密（CTR）+ 完整性（CBC-MAC），合称 AEAD。每帧一次性 nonce，杜绝 WEP 密钥复用。Wi-Fi 4/5/6 默认。

TKIP Temporal Key Integrity Protocol

WPA1 过渡协议，RC4 + 每包密钥混合 + Michael MIC。2012 废弃。新固件不要支持。

PSK Pre-Shared Key · 预共享密钥

WPA2-Personal（家庭）认证。密码 + SSID 经 PBKDF2（4096 轮）派生 PMK。安全性依赖密码强度——抓 4-Way Handshake 可离线爆破。

PMK Pairwise Master Key · 成对主密钥

STA↔AP 共享的根密钥（32B），不在空中传。来源：PSK 派生 或 802.1X/EAP 的 MSK 截取。PMK 本身不加密数据帧，是 4-Way Handshake 的输入。

PTK Pairwise Transient Key · 成对临时密钥

4-Way Handshake 派生的本次会话单播加密密钥。输入：PMK + ANonce + SNonce + 双方 MAC。每次关联重派生。

由三段组成：KCK（签 EAPOL MIC）+ KEK（加密 GTK）+ TK（加密数据帧）。

GTK Group Temporal Key · 组临时密钥

AP 为整个 BSS 生成的组播/广播加密密钥，所有 STA 共用。M3 里用 KEK 加密下发。

为什么单播要独立 PTK：每对密钥独立，一个被破不连累其他 STA；组播必须共用才能让所有人解。

4-Way Handshake 四次握手

RSNA 核心，4 个 EAPOL-Key 帧派生并确认 PTK + 下发 GTK：

M1 AP→STA: ANonce

M2 STA→AP: SNonce + MIC（证明算出 PTK）

M3 AP→STA: 加密 GTK + RSN-IE + MIC

M4 STA→AP: MIC 确认

关联成功 ≠ 能传数据，要 4-Way Handshake 完成进"已授权"才行。

ANonce / SNonce Authenticator / Supplicant Nonce

4-Way Handshake 的随机数。ANonce 由 AP 生成（M1），SNonce 由 STA 生成（M2）。两者凑齐后双方都能算 PTK。保证 PTK 每次新鲜。

MIC Message Integrity Code · 消息完整性码

用密钥（KCK）对帧内容算的校验码。既是完整性也是身份证明——验证 MIC = 没被篡改 + 发送方持有正确密钥。M2/M3/M4 都带 MIC。

EAPOL EAP over LAN

802.1X 定义的帧格式，承载在 802.11 数据帧 Body 里。EAPOL-Key 装 nonce/MIC/GTK。在加密前明文传（鸡生蛋：密钥还没建立），靠 MIC 保完整性。

802.1X / EAP 端口认证 / 可扩展认证协议

WPA2-Enterprise 身份认证。STA ↔ RADIUS 做 EAP（TLS/PEAP/TTLS），AP 当中间人，成功后 RADIUS 给 AP 发 MSK → 派生 PMK。

比 PSK 强：每用户独立认证、密钥集中管理、支持证书。

SAE Simultaneous Authentication of Equals · 对等同步认证

WPA3-Personal 替代 PSK 的协议（Dragonfly / PAKE）。双方用密码派生 Password Element 做 COMMIT/CONFIRM，派生 PMK。

关键改进：抓包无法离线爆破密码（PMK 高熵 + 每会话新鲜），有前向保密。修复了 WPA2-PSK 最大软肋。

PMF / 802.11w Protected Management Frames · 受保护管理帧

给管理帧（Deauth/Disassoc/部分 Action）加加密保护 + 完整性校验。WPA2 可选，WPA3 强制。

修复 Deauth 攻击——没 PMF 时伪造一个 Deauth 就能把 STA 踢下线，早年最常见攻击。

RSN-IE Robust Security Network Information Element

Beacon/Probe Resp/Assoc Req 里的信息元素，宣告 BSS 安全能力：加密套件（CCMP/TKIP）、认证方式（PSK/802.1X/SAE）、是否要 PMF。关联前用它匹配能力。

PHY / 调制（物理层）

OFDM Orthogonal Frequency Division Multiplexing · 正交频分复用

把数据分到多个频域正交的子载波上并行传输。Δf = 1/T（子载波间隔 = 符号周期倒数），sinc 频谱零点对齐，子载波频谱可重叠而互不干扰。

802.11a/g/n/ac: Δf = 312.5 kHz 802.11ax: Δf = 78.125 kHz

⚠️ OFDM 正交是频域（Δf = 1/T），不要和 QAM 的 I/Q 正交（相位差 90°，单载波内）混淆。

I/Q In-phase / Quadrature · 同相 / 正交

单载波内的两路：I 路 cos(ωt)、Q 路 sin(ωt)，相位差 90°。QAM 调制建立在 I/Q 之上。

这是单载波内的正交，和 OFDM 多子载波间的频域正交是两回事。

QAM Quadrature Amplitude Modulation · 正交幅度调制

同时用 I/Q 两路的幅度和相位承载多个比特。

16-QAM: 4 bit/符号 64-QAM: 6 bit 256-QAM: 8 bit 1024-QAM: 10 bit

阶数越高 → 速率越高 → 需更强 SNR（星座点更密，更难区分）。

BPSK Binary Phase Shift Keying · 二进制相移键控

最简单的相位调制，一个符号传 1 bit（0/π 两个相位）。Wi-Fi 最低速率 MCS0 用 BPSK 1/2。

MCS Modulation and Coding Scheme · 调制与编码方案

0–11 的索引（802.11n/ac/ax），标识一组"调制阶数 + 编码率"。MCS 越高 = 速率越高，需更强 SNR。由自适应速率算法（rate adaptation）动态选择。

MCS0 (BPSK 1/2, ~0 dB SNR) → ... → MCS11 (1024-QAM 3/4, ~36 dB SNR)

GI / CP Guard Interval / Cyclic Prefix · 保护间隔 / 循环前缀

每个 OFDM 符号前插入的冗余，吸收多径时延扩展，避免 ISI。802.11a: 0.8 μs。CP 形式（符号尾部复制到头部）还能保持子载波正交。

PAPR Peak-to-Average Power Ratio · 峰均功率比

信号峰值功率与平均功率的比值。OFDM 因多子载波叠加，PAPR 高 → 功放易饱和、需功率回退。单载波（如 FSK）PAPR 低。OFDM 的代价之一。

ICI Inter-Carrier Interference · 子载波间干扰

OFDM 子载波正交性被破坏（频偏、相位噪声、多径超 GI）导致的子载波互相泄漏。频偏敏感是 OFDM 的代价之一 → WiFi 要 ±20 ppm 晶振。

ISI Inter-Symbol Interference · 符号间干扰

前一个符号的多径时延拖到下一个符号里造成的干扰。OFDM 用 GI/CP 吸收。

EVM Error Vector Magnitude · 误差向量幅度

实测星座点与理想星座点的偏差，衡量发射/接收质量。EVM 越小越好。高阶 QAM（1024-QAM）对 EVM 要求极严。

FHSS Frequency Hopping Spread Spectrum · 跳频扩频

在多个载频间按约定序列快速切换。蓝牙（BR/EDR）的核心 PHY。Wi-Fi legacy 802.11-1997 用过，已淘汰。

GFSK Gaussian Frequency Shift Keying · 高斯频移键控

蓝牙（BR/EDR 和 BLE）的 PHY 调制。单载波、恒包络、低阶。与 Wi-Fi 的 OFDM + QAM 形成对照。

RF / 信号质量

RSSI Received Signal Strength Indicator · 接收信号强度指示

接收机测得的信号功率，单位 dBm。链路预算里"接收信号功率"的实测值。负数，因为接收功率永远 < 1 mW。

读数档：−40 极强 −65 好 −75 可用 −85 边缘 <−95 极限

SNR Signal-to-Noise Ratio · 信噪比

信号功率与噪声功率的比值（dB）。SNR = RSSI − 噪声底。决定可达 MCS。

RSSI 本身没意义，SNR 才是"能不能通"的判据。详见 RF 数学速查表。

PER Packet Error Rate · 包错误率

数据包错误/丢失的比例。灵敏度定义在 PER = 10% 处（即 90% 包能正确解调的最低信号功率）。

EIRP Equivalent Isotropically Radiated Power · 等效全向辐射功率

发射功率 + 天线增益的合成量 = EIRP = Tx Power + Ant Gain。表示向某方向辐射的等效功率。法规对 EIRP 有上限。

FSPL Free Space Path Loss · 自由空间路径损耗

信号在自由空间传播的衰减。FSPL = 20·log₁₀(d) + 20·log₁₀(f) + 32.44（d: km, f: MHz）。频率越高、距离越远，损耗越大。

详见 RF 数学速查表。

QoS / 高级特性

QoS Quality of Service · 服务质量

区分不同业务优先级的机制。Wi-Fi 用 802.11e / EDCA 实现。

EDCA Enhanced Distributed Channel Access · 增强分布式信道接入

802.11e 定义的 QoS 版 DCF。用可变 AIFS + 不同的 CWmin/CWmax 区分 4 个接入类别（AC）：

语音 (Voice) 视频 (Video) 尽力而为 (Best Effort) 背景 (Background)

语音 AIFS 最短、CWmin 最小 → 优先级最高。

蓝牙对照术语（桥梁）

TDMA Time Division Multiple Access · 时分多址

把时间切成帧/槽，由中央调度器分配。蓝牙（BLE connection event）的范式。与 Wi-Fi CSMA/CA 形成根本对照。

piconet piconet（无缩写，希腊语"小网"） · 微微网

蓝牙的基本网络单元：1 master + ≤7 active slave。结构上对应 Wi-Fi BSS，但 master 是仲裁者（调度），AP 不调度。

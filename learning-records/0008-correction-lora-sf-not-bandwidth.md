# 概念纠正：LoRa SF 是扩频因子，不是「扩带宽」

**Status**: active

## 学到的

第 4 课（噪声底/SNR）中，教学方在「桥梁框」里错误地把 LoRa 的 SF（Spreading Factor）描述为「扩带宽」，并写「SF 降=扩带宽 → 速率涨但灵敏度掉」。

**学员立即纠正**：「LoRa 的 SF 不是扩码率吗？」—— 学员正确。

## 纠正内容

LoRa 有两个<strong>独立</strong>参数，教学方上一版课件把它们搞混了：

| 参数 | 含义 | 对灵敏度的影响 |
|---|---|---|
| **BW（带宽）** | 信道宽度 | 决定热噪声底（−174 + 10·log₁₀(B)）。窄 BW → 低噪声底 → 高灵敏度 |
| **SF（扩频因子）** | chirp 扫频速度，决定符号时长 | T_sym = 2^SF / BW。高 SF → 长积分 → 处理增益 → 每 SF +1 灵敏度改善约 2.5–3 dB |

关键公式（来自 CSIR 论文）：
- 比特率 R_b = BW × SF / 2^SF
- 符号时长 T_sym = 2^SF / BW

**LoRa 灵敏度高的核心原因是 SF 的处理增益（积分时间），BW 是次要因素。** 这两者是独立的权衡轴。

## 教学影响

1. **只有 BW 的权衡经验能迁移到 Wi-Fi**：LoRa 缩 BW → 噪声底降 → 灵敏度涨，这和 Wi-Fi「信道宽度自适应」是同一物理。
2. **SF 的权衡经验不能迁移到 Wi-Fi**：Wi-Fi 用 OFDM 子载波 + 编码增益，不用 chirp 扩频。SF 在 Wi-Fi 里没有对应物。
3. **课件已修正**：第 4 课桥梁框已改写为「BW 和 SF 是独立参数」，并加了显式的「纠错记录」框（透明记录错误本身，防止学员记忆混淆）。

## Implications

- ✅ <strong>解锁</strong>：第 4 课 BW/SF 区分已澄清，桥梁框现在准确。
- ⚠️ <strong>警示</strong>：教学方对 LoRa 细节的记忆不可靠。涉及 sub-GHz / LoRa 的具体机制时，必须查证后再写，不能凭记忆。这条警示适用于所有后续课程的「桥梁框」。
- ⚠️ <strong>标定</strong>：学员对 LoRa 参数理解准确（能立即识别 SF 不是带宽），说明其 sub-GHz 协议栈经验是真实的、深入的。后续课程可更放心地用 sub-GHz 做桥梁，但要用对术语。

## Evidence

学员原话：「LoRa 的 SF 不是扩码率吗？」—— 直接、准确。学员用的词是「扩码率」（spreading factor 的中文俗称之一），指向正确的概念。教学方随即查证 TTN、CSIR 论文等权威来源确认学员正确，并修正课件。

**反思**：这正是 LR-0006（不读原书，全部由课程交付）要求「课程必须自洽、准确」的反面教材 —— 既然课程是唯一输入，事实错误会更严重，因为没有书做纠正参考。教学方对非 WiFi 领域（如 LoRa 细节）的记忆必须查证。

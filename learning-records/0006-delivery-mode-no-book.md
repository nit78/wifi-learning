# 交付模式变更：不读原书，全部由课程网页交付

**Status**: active

## 学到的

学员明确表示：「我不希望读原书，而是你通过网页教给我。」

这是教学交付模式的根本性变更，影响所有后续课程设计。

## 变更内容

**之前**：CWNA 原书是「主骨架」，课程是「导读 + 桥梁」，每节课末尾推荐读原书章节巩固。

**现在**：
- CWNA 原书降级为**我的内部参考资料**（我读它组织课程、对标覆盖度和准确性）。
- 所有知识由 `lessons/*.html` 直接、完整交付 —— 课程本身必须自洽，不依赖学员课外阅读。
- 课程末尾**不再推荐读原书章节**。外部资料（Tek primer、论文等）也只能标为「可选扩展，不强求」，且不得是理解后续内容的前提。

## 为什么对后续教学重要

1. **课程必须更完整**：不能再假设「学员会去书上补细节」。每节课要把该讲的讲透，不能甩给「详见第 X 章」。
2. **节奏可能需要调整**：原本「一节课 + 读书巩固」的双轨，变成「课程单轨」。可能需要把某些大主题拆成更多小课，避免单课信息过载。
3. **自检题更重要**：既然没有书的练习题做补充，课程内的检索练习成为唯一的掌握度检验。
4. **RETRACING（复习）机制要靠课程间引用**：没有书做回查工具，课程之间要用 HTML 锚点互相链接，形成可回查的知识网。

## Implications

- ✅ <strong>解锁</strong>：所有后续课程按「自洽交付」标准设计。
- ✅ <strong>解锁</strong>：第 1、2、3 课的「推荐资料」段已统一改为「不必额外读资料 / 可选扩展」。
- ⚠️ <strong>调整</strong>：RESOURCES.md 里 CWNA 书已标为「内部参考，学员不读」。其它外部资源（Tek primer、IEEE 论文）保留但标注「可选」。
- ⚠️ <strong>注意</strong>：RF 基础这类需要反复练习的内容（LR-0005 指出学员 RF 数学从零），更要在课程内提供足够算例和自检，不能依赖书中的练习。

## Evidence

学员原话：「我不希望读原书，而是你通过网页教给我。」语气明确，非试探。

**关联**：本记录与 LR-0005（RF 基础被高估）共同指向一个结论 —— 课程必须**自洽、自足、自检**，不假设学员有任何课外输入。

# 笔记功能组件交付（note-taker.js）

**Status**: active

## 交付内容

学员希望「遇到未知知识点时随手记笔记」。经过方案确认（学员选「右键弹窗 + localStorage」），交付了：

1. **`assets/note-taker.js`** —— 笔记组件，功能：
   - 任何课程页选中文字 → 右键 → 弹窗记笔记
   - 自动捕获选中原文 + 记录来源课程页
   - 存到 localStorage（跨课、跨会话保留）
   - 右上角 📨 按钮：查看 / 搜索 / 删除 / 导出
   - 导出为独立 HTML 文件（备份用）
2. **`assets/style.css`** —— 追加了笔记组件全套样式（FAB 按钮、右键菜单、弹窗、列表、toast、打印隐藏）
3. **5 节课 + 速查表**全部引入 `note-taker.js`
4. **`NOTES-GUIDE.html`** —— 使用说明页（也带笔记功能，可当场试用）

## 技术约束（已告知学员）

浏览器安全沙箱禁止网页直接写本地文件，所以采用 localStorage + 导出方案：
- ✅ 同一浏览器跨课/跨天保留
- ❌ 换浏览器、清缓存、隐私模式会丢 → 需定期「导出 HTML」备份
- ❌ 非真正的本地文件（这是纯前端的固有限制）

## 为什么对后续教学重要

1. **学员有了主动记录工具**：学习时遇到不懂的点、顿悟的点、疑问，可随手记下，不必等我问。
2. **笔记是 retrieval practice 的素材**：学员自己写的理解（用自己的话）是最高价值的复习材料。
3. **笔记内容会反哺教学**：如果学员愿意分享导出的笔记，我能看到他的理解偏差和疑问点，用于调整后续课程。但这是可选的，隐私由学员控制。
4. **后续课程默认带笔记功能**：只需在 HTML 末尾加一行 `<script src="../assets/note-taker.js"></script>`。

## Implications

- ✅ <strong>解锁</strong>：所有现有课程页（1-5 课 + 速查表）已支持笔记。
- ✅ <strong>规则</strong>：后续所有新课程/参考页必须引入 note-taker.js。
- ⚠️ <strong>注意</strong>：localStorage 依赖单一浏览器，导出功能是必要的备份手段，需提醒学员定期导出。

# RTS/CTS

是IEEE 802.11无线网络中用于解决隐藏节点问题和减少数据碰撞的一种信道预约机制。它通过在数据传输前增加握手过程，确保信道在发送期间不被其他节点占用，从而提升传输可靠性。

核心流程如下：

	发送方A 先发送一个短小的 RTS帧，包含即将发送的数据长度信息。

	接收方B 收到RTS后，回复 CTS帧，通知A可以发送，同时广播给周围节点在指定时间内保持静默（NAV字段指示）。

	发送方A 接收到CTS后，在预约时间窗口内发送数据帧。

	其他节点根据CTS中的NAV值暂停发送，避免冲突。
	
	数据传输完成后，信道重新进入竞争状态，继续通过 CSMA/CA 机制争夺使用权。

## RTS阈值（RTS Threshold） 是控制该机制触发条件的关键参数：

	数据帧长度 > 阈值 → 启动RTS/CTS握手。

	数据帧长度 ≤ 阈值 → 直接使用CSMA/CA，减少额外开销。 默认推荐值为 1400字节，主要保护大包传输，因为大包在空口传输时间长，更易受干扰。

## 典型应用场景：

	隐藏节点环境：如多障碍物或覆盖不完整的网络。 ->我不知道对方，但对方能收到

	高密度网络：大量设备集中且频繁通信时减少碰撞。 ->竞争代价太大

	长数据帧传输：如视频流、文件传输等。 ->太长一定会冲突

	高可靠低延迟需求：如VoIP、实时监控。

事实核查 —— 课件哪里写错了

这是一节特殊的补充课（主线课编号不受影响，下一主线课仍是第 8 课省电模式）。它不教你新概念，而是把第 1–7 课的内容拿去对照网络权威资料（IEEE 802.11 标准原文、CWNA、CWNP 白皮书、FCC/ETSI 法规、NI / Qualcomm 一手文档），把不够准、甚至写错的地方摊开。

为什么要这样做？因为你的使命（见 MISSION）是转型 WiFi 固件工程师，而学习记录 LR-0006 规定「课程是唯一输入，不读原书」。这意味着：课件写错的，就是你会当事实记住的。所以这一课的目标是——把地基里松动的砖挑出来，换上标准原文的砖。

本课支点 / Core Insight

核查结论分三档：7 项核对通过（可放心复用）、1 处实质错误（RF 链路预算漏了 Noise Figure）、4 处精度问题（SIFS 拆解数字、法规功率标签、算例 1 dB 偏差、quiz 标题矛盾）。最该修的是 RF 那处——它影响你对灵敏度和 SNR 的整个心智模型。

先说对的：7 项通过核对

先给你吃颗定心丸——下面这些是课件里写得准的，经得起一手标准核对，可以放心复用、放心记。

✅ 通过 · BEB 退避

第 5 课：BEB 公式与 CW 序列

lessons/0005-csma-ca-backoff.html

CW = 2ⁿ·(CWmin+1) − 1，序列 15→31→63→127→255→511→1023（封顶），与 IEEE 802.11 DCF 标准逐项吻合。标准递推是 CW_new = 2·(CW_old+1) − 1，闭式解正是课件这个。CWmin=15/CWmax=1023 是 OFDM PHY（802.11a）的取值——这点课件标对了（注意：802.11b DSSS 是 CWmin=31，课件没误用）。

✅ 通过 · DCF 时序

第 5 课 / 速查表：SIFS / SlotTime / DIFS

lessons/0005, reference/wifi-terms-cheatsheet.html

802.11a：SIFS 16 / Slot 9 / DIFS 34 μs（16+2×9=34 ✓）。802.11b：SIFS 10 / Slot 20 / DIFS 50 μs（10+2×20=50 ✓）。全部符合标准。换算关系 DIFS = SIFS + 2×SlotTime 正确。

备忘（非错误）：802.11g 默认兼容模式其实复用 802.11b 时序（slot 20 μs），只在 ERP-OFDM-only 模式才切到 9 μs。课件只拿 802.11a/b 做对照，没踩这个坑。

✅ 通过 · 子载波结构

第 2 课：802.11a 子载波布局

lessons/0002-ofdm-orthogonality.html

20 MHz ÷ 312.5 kHz = 64 点 FFT，其中 52 个有效（48 数据 + 4 导频），其余 12 个为空（左右保护带 + 中心 DC null）。与 IEEE Std 802.11a-1999 §17.3.5、Keysight/Tek/CWNP 一致。

✅ 通过 · OFDM 正交性

第 2 课：Δf = 1/T、sinc 零点、两种正交之分

OFDM 频域正交（Δf=1/T，sinc 零点对齐）vs QAM 的 I/Q 正交（相位差 90°，单载波内）——区分得清楚、正确。

✅ 通过 · 状态机与帧结构

第 6、7 课：State 1/2/3、四地址、AID、Auth

三态状态机 + Class 1/2/3 帧、Open System 两帧 Auth（algorithm=0，几乎不验证身份）、AID 范围 1–2007、ToDS/FromDS 四地址映射——全部与标准一致。「802.11 Authentication ≠ 安全认证」这个纠正尤其重要且正确。

✅ 通过 · Beacon 间隔

第 7 课 / 速查表：Beacon 周期 102.4 ms

默认 100 TU，1 TU = 1024 μs = 1.024 ms，所以 100 × 1.024 = 102.4 ms。正确。CWNP 博客专门讲过这个「常被当成 100 ms」的细节。

✅ 通过 · piconet 结构

术语表 / 速查表：piconet 1 master + ≤7 active slave

标准蓝牙 BR/EDR piconet：1 master + 最多 7 active slave（受 3-bit AM_ADDR 限制），另有最多 255 parked 设备（8-bit PM_ADDR）。课件写的是 active slave 上限，准确。

实质错误：SNR / 链路预算漏了 Noise Figure

这是这次核查发现的最该修的地方。第 4 课和 RF 速查表用热噪声底 −101 dBm直接算 SNR：

课件当前的写法（有问题）/ As taught SNR(dB) = RSSI(dBm) − N_thermal(dBm)
例：RSSI −49 dBm − (−101 dBm) = +52 dB

但所有权威 RF 工程来源都用接收机噪声底 = kTB + Noise Figure (NF)：

正确的写法 / Authoritative form Sensitivity(dBm) = −174 + 10·log₁₀(B) + NF + SNR_required
接收机噪声底 = kTB + NF　（不是裸 kTB）

Wi-Fi 典型 NF = 5–10 dB：IEEE 标准灵敏度推导假设 10 dB，CEPT ECC Report 288 用 10 dB，手机 STA 因紧凑封装（天线开关 + 滤波器插损顶在 LNA 前）约 7–10 dB，AP 约 5–7 dB。课件相当于把 NF 当成 0 dB，于是 SNR 被系统性夸大 5–10 dB。

更糟的是，这个遗漏暴露了课件自己的内部矛盾——你能用课件给的数字反过来证伪课件：

❌ 自相矛盾

MCS0：「需 ~0 dB SNR」vs「灵敏度 −93 dBm」

课件（速查表 §⑤）写 MCS0 需 ~0 dB SNR，灵敏度 −93 dBm，噪声底 −101 dBm。但按课件自己的公式：−93 − (−101) = +8 dB。这 8 dB 不是 0。

这 8 dB 正好等于 NF(~5–6 dB) + 解调 SNR(~2 dB)。换句话说，课件把两个分量都丢了，但因为它也把灵敏度数字调成了「看起来对」的值，矛盾被表面掩盖了。

真实的 MCS0 要求（一手来源核对）：

  • 解调 SNR（在接收机噪声底 kTB+NF 之上）：约 2 dB。来源：[RF Essentials](https://rfessentials.com/rf-knowledge-base/what-is-the-sensitivity-requirement-for-a-wi-fi-7-receiver-at-the-maximum-data-r/) 明确给 ~2 dB。无编码 BPSK @ BER 10⁻⁵ 约 9.6 dB（[Princeton COS598A](https://www.cs.princeton.edu/courses/archive/spring17/cos598A/lectures/L3.pdf)），802.11 的 rate-½ 卷积码给 ~5–7 dB 编码增益 → 落到 ~2–4 dB。
  • RSSI 高于热噪声底（课件实际算的那个量）：约 7–10 dB（= NF 5–8 + 解调 SNR 2）。

另外，IEEE 802.11n/ac MCS0 @ 20 MHz 的最低灵敏度标准是 −82 dBm（[NI WLAN 测试 PDF](http://download.ni.com/evaluation/rf/Introduction_to_WLAN_Testing.pdf) 的表），不是 −93；−93 更接近 802.11ax HE20 MCS0。课件没标 PHY 代际，把不同代的数字混着用了。

关键的反直觉点

你以后读芯片 datasheet 的灵敏度表（比如 −93 dBm @ MCS0），那个数字已经把 NF 和解调 SNR 都算进去了。它就是「这台接收机在这个速率上能解调的最低 RSSI」。所以实战中你不需要自己算 NF——直接拿 datasheet 灵敏度表对照 RSSI 就行。课件第 3 课「RSSI 对照灵敏度表判 MCS」的思路是对的；错的是第 4 课额外加的那层「SNR = RSSI − 热噪声底」——那一步缺了 NF，把简单的事搞歪了。

来源： [RF Essentials — Noise Floor/BW/NF 关系](https://rfessentials.com/rf-knowledge-base/what-is-the-relationship-between-noise-floor-bandwidth-and-noise-figure-in-a-rec/) · [RF Essentials — Wi-Fi 7 灵敏度要求（MCS0 SNR≈2 dB）](https://rfessentials.com/rf-knowledge-base/what-is-the-sensitivity-requirement-for-a-wi-fi-7-receiver-at-the-maximum-data-r/) · [Marki Microwave — NF 与灵敏度](https://markimicrowave.com/technical-resources/application-notes/noise-figure-receiver-sensitivity-guide/) · [Rohde & Schwarz — 接收机 NF](https://www.rohde-schwarz.com/us/applications/examining-receiver-noise-figure_56279-1170880.html) · [ECC Report 288（CEPT，Wi-Fi 典型 NF 10 dB）](https://docdb.cept.org/download/4d5f88b8-7f6d/ECC%2520Report%2520288.pdf) · [NI — WLAN 测量入门（MCS0 = −82 dBm）](http://download.ni.com/evaluation/rf/Introduction_to_WLAN_Testing.pdf)

怎么修（二选一）

(a) CWNA 经验路线：噪声底用实测值（如频谱仪测的 −95 dBm），SNR = 信号 − 实测噪声底，灵敏度用厂商表。NF 隐含在实测噪声底和灵敏度数字里。这条路简单、贴近实战，CWNA 就是这么干的。

(b) RF 工程严格路线：保留 kTB 公式，但加 NF 项：Sensitivity = −174 + 10log(B) + NF + SNR_req，给 NF 一个值（AP 5–7 dB / STA 7–10 dB），给 MCS0 SNR_req ≈ 2 dB。这条路精确、适合固件 RF/PHY 调参场景。

别再两头不靠——当前课件借了 kTB 物理公式，又丢了 NF 项，是两种路线的混合体。

精度问题：SIFS 四分量拆解数字不是标准值

第 5 课的「教学坦白」框、术语速查表、术语表 都写了 SIFS 的四分量拆解：

课件当前写法 / As taught aSIFSTime = aRxRFDelay + aRxPLCPDelay + aMACProcessingDelay + aRxTxTurnaroundTime
802.11a 拆解：RxRF ~0.5 μs + PLCP 解码 ~12.5 μs + MAC ~0.5 μs + RxTx ~2 μs = 16 μs
✅ 公式对

四分量公式本身是标准原文

这个公式逐字见于 [CWNP《802.11 Arbitration》白皮书](https://www.cwnp.com/uploads/802-11_arbitration.pdf)、IEEE TGj 的 comment-resolution 记录、Rutgers WINLAB 的 802.11 MAC 字典。形式正确。

✅ 核心论点对

「接收机处理占大头，收发切换是小头」

这个论点正确且重要——它纠正了「SIFS 就是收发切换时间」的常见误解。RxTx 切换标准上界 < 2 μs，只占 16 μs 的 ~12%；接收机处理（RF + PLCP）才是大头（~75–88%）。这也是为什么 802.11a 的 SIFS（16 μs）比 802.11b（10 μs）长——OFDM 接收机要做 AFC、FFT、串并转换、Viterbi 译码，DSSS 接收机简单得多。

⚠️ 数字需修

「0.5 / 12.5 / 0.5 / 2 μs」不是标准值

IEEE Table 93（OFDM PHY 特性表）只钉死总和 16 μs + 两个上界：aRxTxTurnaroundTime < 2 μs、aMACProcessingDelay < 2 μs。而 aRxRFDelay / aRxPLCPDelay 标的是「Implementation dependent」——标准不给他们固定数值。

课件那组 0.5 / 12.5 / 0.5 / 2 实际来自 [专利 WO2015186941A1](https://patentimages.storage.googleapis.com/04/3e/0a/34a7fb5624f67a/WO2015186941A1.pdf) 的一个示例，不是标准值。

更权威的拆解来自 [IEEE 802.11-99/062](https://www.ieee802.org/11/Documents/DocumentArchives/1999_docs/906272W-LB17-Comment-802.pdf)（NTT 的 Morikura 在 1999 年定稿时的意见）：接收机处理 ~14 μs = AFC 4 μs + 串并转换 4 μs + FFT 4 μs + 译码 2 μs。[NI 的参考实现](https://www.ni.com/en/support/documentation/supplemental/16/labview-communications-802-11-application-framework-2-0-and-2-0-.html)实测 D1 ≈ 12 μs。

所以「78%」「12.5 μs PLCP」这个量级站得住，但「12.5 μs PLCP」把 FFT、串并转换都算进 PLCP 了——FFT 是数字前端，不严格属于 PLCP。更准的说法：接收机侧处理（含 AFC、FFT、译码）是大头（~12–14 μs）。

怎么修

把「0.5/12.5/0.5/2」标为「典型/示意值」，并改引 IEEE 99/062 的推导（接收机处理 ~12–14 μs，含 AFC/FFT/译码），或至少注明「此拆解为专利示例，标准只规定总和与上界」。核心论点（接收机处理是大头）保留不动。

精度问题：法规功率标签贴错频段

RF 速查表 §③ 的 dBm↔mW 对照表里，两条法规标注都不对：

课件标签课件含义权威实际值问题
+23 dBm = 200 mW 「2.4 GHz 法规上限附近」 FCC 2.4G：30 dBm 传导 / 36 dBm EIRP；ETSI 2.4G：20 dBm EIRP（100 mW） 23 dBm 既离 FCC 远，又超 ETSI 3 dB（欧洲不合规）。23 dBm 其实是 ETSI 5 GHz 室内限值，被错贴到 2.4G。
+30 dBm = 1 W 「5 GHz 室外 AP 上限」 FCC U-NII-3 室外：30 dBm 传导 / 36 dBm EIRP；ETSI 5G 室外：基本不允许 >23 dBm 仅对 FCC 成立，且室外更该看 36 dBm EIRP。ETSI 无此数。30 dBm 又恰好等于 FCC 2.4G 传导限——等于把「FCC 2.4G 传导数」贴给了「5G 室外」。

来源： [FCC §15.247（2.4 GHz）](https://www.ecfr.gov/current/title-47/chapter-I/subchapter-A/part-15/subpart-C/subject-group-ECFR2f2e5828339709e/section-15.247) · [FCC Part 15 Subpart E（U-NII / 5 GHz）](https://www.ecfr.gov/current/title-47/chapter-I/subchapter-A/part-15/subpart-E) · [ETSI EN 300 328（2.4 GHz，EIRP 20 dBm）](https://www.etsi.org/deliver/etsi_en/300300_300399/300328/02.00.20_20/en_300328v020020a.pdf) · [ETSI EN 301 893（5 GHz，室内 23 dBm）](https://www.etsi.org/deliver/etsi_en/301800_301899/301893/02.02.01_60/en_301893v020201p.pdf)

怎么修

2.4 GHz：FCC 传导 30 dBm / EIRP 36 dBm；ETSI EIRP 20 dBm（100 mW）。
5 GHz 室外：FCC U-NII-3 传导 30 dBm / EIRP 36 dBm；ETSI 以室内为主，23 dBm。
法规是分地区 + 分频段 + 分室内外的，速查表里一行讲不清，建议要么删掉法规列、要么标「典型值，细节查当地法规」。

小问题：算例差 1 dB + quiz 标题矛盾

⚠️ 1 dB 偏差

速查表 §⑦A 与课程正文不一致

rf-math-cheatsheet §⑦A 写「近距离 5 GHz, 20 m → RSSI −48 dBm → SNR +53 dB」。但第 4 课和第 3 课正文同一场景算的是 RSSI −49 dBm → +52 dB。同一工作区内 1 dB 漂移，应统一（建议都以 −49 dBm 为准，因为第 3 课的链路预算明细算出来的就是 −49）。

⚠️ quiz 标题与正文打架

第 4 课 quiz：「Wi-Fi 比 LoRa 差 30 dB 因带宽」

lessons/0004-noise-floor-snr.html（quiz 题 2）

题干归因「带宽」，但选项解释（22 dB 带宽 + 8 dB 调制）是对的，而且本课自己的「纠错记录」框（对应 LR-0008）已写明「LoRa 灵敏度高的核心是 SF 的处理增益，BW 是次要因素」。标题与正文/纠错框自相矛盾。应把标题改成「带宽 + 调制/SF 处理增益」。

方法论：CWNA 经验路线 vs RF 工程路线

这次核查顺带暴露一个更深的方法论问题，值得单独记一笔。

直接抽取工作区根目录的 CWNA-108（第 6 版）PDF 全文搜，「noise figure」零命中。也就是说：

  • CWNA 走经验路线：噪声底是实测/环境值（如「2.4 GHz ISM 信道典型 −100 dBm，工厂 −90 dBm」），SNR = 信号 − 实测噪声底，灵敏度是厂商每速率门限表，链路预算用「Rule of 10s and 3s」近似。不教 kTB 物理公式。
  • RF 工程走严格路线：kTB + NF + 解调 SNR，每个项都要算。
  • 当前课程两头不靠：借了 kTB 物理公式（CWNA 不用），又丢了 RF 工程的 NF 项 → 既不简单也不严格。
给你的建议（作为固件工程师）

你是协议栈/驱动方向，不是 RFIC 方向。所以实战里走 CWNA 经验路线就够——读 datasheet 灵敏度表、用工具测噪声底，不需要手算 NF。kTB 公式留作「理解为什么带宽影响灵敏度」的物理直觉即可（这点第 4 课的带宽对比讲对了）。第 4 课的「SNR = RSSI − 热噪声底」那层算式建议弱化或删掉，因为它既不准（漏 NF）也不是你日常要做的（你直接查表）。真正对你有用的是：RSSI 实测 → 对照 datasheet 灵敏度表 → 判 MCS。这条第 3 课已经讲对了。

这课的钉子：不迷信课件，也不迷信单点资料

这节核查课的钉子不是某个具体数字，而是一个习惯：

  • 课件是脚手架，不是真理。LR-0006 让课件成为唯一输入，这放大了错误的影响——所以更要勤核对。
  • 核对要找一手：IEEE 标准原文 > 标准制定过程记录（如 802.11-99/062）> CWNP 白皮书 / NI 参考实现 > 厂商 datasheet > 博客。越靠前越权威。
  • 数字会漂：课件里这次发现 1 dB 偏差、法规标签错位、SIFS 拆解用了专利示例——都是「看起来对、细查有出入」的典型。养成「存疑就标 [待核对]」的习惯。
  • 最该修的是 SNR/NF 那处，因为它直接污染你对「灵敏度怎么来的」这个核心模型的理解。其余是精度问题，不伤根本。

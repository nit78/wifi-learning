CSMA/CA 退避 —— 分布式竞争怎么不打架

RF 数学告一段落，进入协议主线 —— 这是你的强项区。第 1 课埋了一根钉子：「Wi-Fi 是分布式竞争，不像蓝牙的调度式 TDMA（时分多址）」。那课只讲了范式，这课讲机制：分布式竞争具体怎么操作，靠什么尽量避免碰撞。

你做过协议栈，对「状态机、时序、帧交换」很熟。所以这课用协议栈视角切入 —— CSMA/CA（载波侦听多址接入 / 碰撞避免）本质就是一个 MAC（介质访问控制层）状态机 + 一组时序规则。你脑子里可以一直对照：「这步在我做的 BT/sub-G 协议栈里对应什么」。

支点：CSMA/CA = 先听 + 等随机时间 + 发

本课支点 / Core Insight

CSMA/CA 的全部机制可以压缩成一句话：想发 → 先听信道空闲一段时间（DIFS，DCF 帧间间隔）→ 再等一个随机退避（backoff）→ 然后发。DIFS 是固定的「准入门槛」，backoff 是随机的「错峰手段」。两者一起，让一堆无中心的 STA 尽量不撞车。

和 BT 的根本对照：BT 由 master 分配 slot，时间表是定的；Wi-Fi 没有 master 发时刻表，每个 STA（站点）自己算一个随机时间等。这就是「调度式 vs 竞争式」在 MAC 层的具体兑现。

时序图：一次成功的发送长什么样

先把一次成功的帧发送画出来。这是整个 DCF（Distributed Coordination Function，分布式协调函数）的心脏：

时间 → STA 想发数据 │ ├─ 信道忙？是 → 等 │ 信道闲？→ 继续监听 DIFS（34 μs，5 GHz） │ ↓ │ DIFS 内一直闲？是 → 进入 backoff │ ↓ │ 随机 backoff 计数（比如 5 个 slot） │ 每个 slot（9 μs）信道闲 → 计数 −1 │ ↓ │ 计数 = 0 → 发 DATA 帧 │ ↓ │ 等 SIFS（短帧间间隔，16 μs） │ ↓ │ 收到 AP（接入点）的 ACK (确认帧) ✓ 成功 │ ↓ │ CW 重置回 CWmin (最小竞争窗口)，本次结束 │ └─ 若 DIFS 内信道变忙 → 重新等 DIFS（不递减 backoff） 若 backoff 中信道变忙 → 冻结计数，等信道闲 + DIFS 后继续递减

三个要点，记牢：

  1. DIFS 是「准入门槛」：必须先连续空闲 DIFS（34 μs），才有资格进入 backoff。这一步过滤掉「刚刚有人发完」的时段。
  2. backoff 是「随机错峰」：从 [0, CW] 里随机选一个数，按 slot 递减。多个 STA 即使同时通过 DIFS，随机数不同，大概率不会同时发。
  3. SIFS 比 DIFS 短：所以 ACK（用 SIFS）一定比新数据（用 DIFS）先抢到信道。进行中的对话优先级高于新对话。

IFS 家族：用时间间隔做优先级

上面的 DIFS、SIFS 都是 IFS（Interframe Space，帧间间隔）。802.11 用不同长度的 IFS 实现优先级 —— 短 IFS = 高优先级。这是协议栈工程师应该觉得眼熟的设计（类似中断优先级）。

IFS802.11a (5 GHz)802.11b (2.4 GHz)用途 / 优先级
SIFS 16 μs 10 μs 最高。ACK、CTS (允许发送)、继续已开始的帧交换
PIFS (PCF 帧间间隔) 25 μs 30 μs 中。AP 轮询用（PCF，实际很少用）
DIFS 34 μs 50 μs 普通。普通数据帧发起接入
EIFS (扩展帧间间隔) 更长 更长 收到错误帧后用（等更久，避免踩到看不懂的交换）
AIFS (仲裁帧间间隔) 可变 可变 QoS (服务质量) 用，802.11e / EDCA (增强分布式信道接入) 按业务类型分优先级
IFS 的换算关系 SIFS < PIFS = SIFS + 1×SlotTime（时隙） < DIFS = SIFS + 2×SlotTime
桥梁 / From BT

你在 BT 协议栈里调过 T_IFS（帧间间隔）。BT 的 IFS 主要是「保证收发转换的物理时间」，单一值。Wi-Fi 把 IFS 分成多个等级，用来做接入优先级 —— 这是竞争式系统特有的：没有 master 调度优先级，只能靠「谁等得短谁先发」来区分。

Backoff 与 CW：随机错峰的数学

backoff 计数器从 CW（Contention Window，竞争窗口） 里随机取。CW 不是固定的，会随碰撞动态变化 —— 这就是 BEB。

具体规则：

  1. 首次发送：CW = CWmin = 15（802.11a）。从 [0, 15] 随机选一个数（比如 7）。
  2. 按 slot 递减：每过一个空闲 slot（9 μs），计数 −1。信道忙时冻结，不递减。
  3. 计数到 0 → 发送。
  4. 收到 ACK → 成功，CW 重置回 CWmin = 15。
  5. 没收到 ACK（超时）→ 判定碰撞，CW 翻倍，重选 backoff，重试。
BEB（二进制指数退避）公式 CW = 2ⁿ · (CWmin + 1) − 1
n = 重试次数，封顶 CWmax (最大竞争窗口) = 1023

看 CW 怎么随碰撞增长（802.11a）：

重试次数 nCW随机取值范围 [0, CW]平均等待
0（首次） 15 [0, 15] 8 slot
1 31 [0, 31] 16 slot
2 63 [0, 63] 32 slot
3 127 [0, 127] 64 slot
... ... ... ...
6（封顶） 1023 [0, 1023] 512 slot
BEB 的精妙之处

碰撞越多，CW 越大，等待越久。这像自助调节的「退让」：信道拥挤时大家自动拉长退避，给彼此更多空间；空闲时 CW 回到 15，快速发送。这是无中心的自适应拥塞控制，不需要 AP 介入。

对比 BT：BT 的 slot 由 master 定，不需要这种自适应 —— 调度式系统天然不碰撞。

为什么还会碰撞？（第 1 课的悬案）

第 1 课我问过：「CSMA/CA 先听后说，为什么还会碰撞？」现在你能看懂两个答案了：

原因 1：退避撞车（概率碰撞）

两个 STA 同时通过 DIFS，进入 backoff。如果它们随机数恰好相同或接近，计数同时归零 → 同时发送 → 碰撞。这是概率事件，无法完全消除，只能靠 BEB 在多次碰撞后把 CW 拉大，降低再次撞车的概率。

原因 2：隐藏节点（hidden node）

这是 CSMA/CA「先听后说」结构性无能为力的场景：

(听不到彼此) STA-A ◄──── 听不到 ────► STA-C \ / \ A 发时 C 听不到 / \ / \ C 发时 A 也听不到 / \ / \ / \ 都能到 AP / ▼ ▼ ┌──────────┐ │ AP │ ← A、C 同时发 → AP 处碰撞 └──────────┘

A 发给 AP 时，C 因为听不到 A，CCA（信道空闲评估）判断信道闲 → C 也发 → 在 AP 处撞。A 和 C 互相隐藏，所以叫隐藏节点。「先听后说」在这失效，因为听不到的对方恰恰就是会撞你的人。

缓解：RTS/CTS 握手

802.11 为隐藏节点设计了 RTS/CTS。A 想发数据前，先发一个短的 RTS (请求发送) 给 AP；AP 回 CTS（用 SIFS，高优先级）。CTS 是广播的，A 和 C 都能听到。C 听到 AP 说「我要收 A 的数据」，就知道该退避。代价：每次发送多了两个控制帧，小数据不划算，所以通常只对大数据帧启用（有阈值 RTS threshold）。

对驱动栈：这套机制落在代码哪里

对驱动重要 / For Driver Stack

这套 CSMA/CA 机制大部分在固件/硬件里实现（CCA、backoff 计时、SIFS 响应都是微秒级，软件来不及），驱动栈层面你主要打交道的是：

  • tx 路径：把帧交给固件排队，固件按 DCF 规则发送。你看到的是「提交后等多久 TX 状态回来」—— 这个延迟里就包含 DIFS + backoff。
  • 重传与 retry count：没收到 ACK 时，固件按 BEB 重试，达到 retry limit 上报失败。驱动层看到的「重传次数」反映信道拥塞程度。
  • QoS / WMM（EDCA）：不同 AC（语音/视频/尽力而为/背景）用不同 AIFS 和 CW 范围。驱动按业务类型分队列，固件按 AIFS 接入。这是 802.11e 的内容，后续课展开。
  • 调试信号：连续 high retry count / 低 ACK 率 → 信道拥挤或隐藏节点；高 PER + 正常 retry → RF 问题（回看第 4 课 SNR）。

自检：协议主线立稳了吗

这课的钉子：第二根钉子立起来了

现在你有两根钉子了：

  • 钉子 1（PHY，物理层）：OFDM 频域正交（第 2 课）—— 决定「信号怎么传」
  • 钉子 2（MAC）：CSMA/CA 分布式竞争（本课）—— 决定「谁什么时候传」

这两根钉子 + RF 工具箱（速查表），构成了 Wi-Fi 的物理层和介质访问层的核心。之后的所有协议课（帧结构、关联流程、QoS、漫游、安全）都挂在这两根钉子上。

你应该现在能在脑子里回答：

  • 一次成功发送的完整时序（DIFS → backoff → DATA → SIFS → ACK）
  • 为什么 ACK 用 SIFS（短 IFS = 高优先级，保护进行中的对话）
  • BEB 怎么随碰撞增长（CW 倍增，封顶 1023）
  • 两个碰撞原因（退避撞车 + 隐藏节点）和 RTS/CTS 的作用

噪声底与 SNR（信噪比）—— 为什么 Wi-Fi 灵敏度比蓝牙差

上一课你掌握了 dB/dBm 和链路预算，能算出 RSSI（接收信号功率）。但这只回答了「信号有多强」，没回答「够不够」。够不够，要对照的不仅是灵敏度表 —— 更深层的问题是「背景有多吵」。这就是噪声底。

这课从你刚掌握的 dB 工具出发，补上 RF 链路预算的另一半：RSSI → SNR → MCS。然后用一个具体问题把它焊死 ——

你做 sub-GHz 灵敏度能到 −120 dBm，Wi-Fi 只到 −95 dBm 左右。Wi-Fi 接收机更差吗？还是另有原因？

答案藏在「带宽」里。而带宽之所以决定噪声，是因为一条物理定律：热噪声。这课把这条定律讲透，顺便把第 2 课 OFDM（正交频分复用）的「宽带代价」、第 3 课的 dB 工具，全部焊在一起。

支点：SNR 才是「能不能通」的真正判据

本课支点 / Core Insight

信号强弱（RSSI）本身没意义，有意义的是「信号比噪声强多少」—— 这就是 SNR。而噪声由带宽决定（带宽越宽，进入的噪声越多），所以宽带系统天生噪声底高、灵敏度差。Wi-Fi 灵敏度不如 sub-GHz，不是技术差，是物理。

SNR = Signal-to-Noise Ratio，信噪比。注意是「比」（比值，所以是 dB），不是绝对量。

先建直觉：噪声从哪来

想象你在嘈杂的咖啡馆里听朋友说话。朋友的声音是「信号」，周围人的嘈杂是「噪声」。能不能听清，不取决于朋友声音多响，而取决于声音比嘈杂声强多少。这就是 SNR。

无线电里的「嘈杂」有两个来源：

  1. 热噪声（thermal noise）：导体里电子随机热运动产生的噪声。这是物理底，无法消除。它和温度、带宽有关。
  2. 干扰（interference）：其它发射机（微波炉、蓝牙、邻居 AP）在这个频段发的能量。这个可以靠设计减轻（换信道、定向天线、跳频）。

这课只讲热噪声 —— 因为它是物理底，定义了「再怎么优化也到不了」的极限。干扰是后续课（信道规划、CSMA/CA）的内容。

热噪声公式：一条定律，记住一个常数

热噪声功率由带宽决定，公式很短：

热噪声功率 / Thermal noise power（室温 ≈ 290 K） N(dBm) = −174 + 10 · log₁₀( B(Hz) )

整条公式你只需要记住一件事：−174 dBm/Hz。这是 RF 工程师的「光速常数」—— 任何带宽的热噪声底，都是 −174 加上带宽的对数项。

−174 怎么来的（可选，不理解也不影响）

热噪声功率 = k·T·B（k 玻尔兹曼常数，T 开尔文温度，B 带宽）。室温 290 K 下，每 Hz 的热噪声功率约 4×10⁻²¹ W，换算成 dBm 就是 10·log₁₀(4×10⁻²¹ / 10⁻³) ≈ −174 dBm。这是 k 和 T 决定的物理常数，跟具体技术无关。

代入几个实际带宽感受一下 —— 注意每行怎么从 −174 加出来的：

热噪声底计算（从 −174 出发） 基准： −174 dBm/Hz LoRa 125 kHz： −174 + 10·log₁₀(125000) = −174 + 51 = −123 dBm 蓝牙 1 MHz： −174 + 10·log₁₀(1000000) = −174 + 60 = −114 dBm Wi-Fi 20 MHz： −174 + 10·log₁₀(20000000)= −174 + 73 = −101 dBm Wi-Fi 80 MHz： −174 + 10·log₁₀(80000000)= −174 + 79 = −95 dBm Wi-Fi 160 MHz： −174 + 10·log₁₀(1.6×10⁸) = −174 + 82 = −92 dBm

看出规律了吗？带宽每 ×10，噪声底 +10 dB。这是 log 的直接后果（10·log₁₀(10) = 10）。所以：

  • LoRa 125 kHz → 蓝牙 1 MHz：带宽 ×8，噪声底 +9 dB
  • 蓝牙 1 MHz → Wi-Fi 20 MHz：带宽 ×20，噪声底 +13 dB
这就是 Wi-Fi 灵敏度比蓝牙差的全部原因

Wi-Fi 带宽是蓝牙的 20 倍 → 噪声底高 13 dB → 灵敏度必然差约 13 dB。不是 Wi-Fi 接收机差，是宽带系统天生噪声底高。这是第 2 课 OFDM「宽带代价」在数字上的具体兑现。

SNR：把 RSSI 和噪声底焊在一起

有了 RSSI（第 3 课）和噪声底（本课），SNR 就是两者相减（dB 域）：

SNR 的定义 / SNR definition SNR(dB) = RSSI(dBm) − N_noise(dBm)

用第 3 课那道链路预算的 RSSI = −49 dBm，算一下 SNR：

SNR 计算（20 MHz Wi-Fi） RSSI（第 3 课算出） = −49 dBm 噪声底（20 MHz 热噪声） = −101 dBm SNR = −49 − (−101) = −49 + 101 = +52 dB 含义：信号比噪声强 52 dB（约 16 万倍）→ 极强 SNR

52 dB 是什么概念？对照 SNR → MCS 表（第 3 课给过灵敏度版，这里是 SNR 视角）：

MCS调制所需 SNR（约）含义
MCS0 BPSK 1/2 ~0 dB 信号只比噪声强一点点就能收
MCS3 16-QAM 1/2 ~9 dB 中速
MCS7 64-QAM 3/4 ~21 dB 高速，需清晰信号
MCS9 256-QAM 5/6~29 dB Wi-Fi 5 最高速，需很强 SNR
MCS11 1024-QAM 3/4~36 dB Wi-Fi 6 最高速，需极强 SNR

52 dB 的 SNR 超过 MCS11 的需求（36 dB），所以这种近距离场景能用最高速。但注意：MCS11 以上就没有了 —— SNR 再高也用不上，因为调制阶数到顶了。这就是为什么你贴着 AP 测速，到了某个值就不再涨。

把整条链路串起来：一道完整算例

现在把第 3 课 + 本课的所有东西串成一条完整的算式。场景换成远距离，感受 SNR 怎么决定一切：

完整链路预算 + SNR + MCS 判定（5 GHz, 40 m, 20 MHz） ── 第 3 课：算 RSSI ── ① 发射功率 +15 dBm ② 路径损耗 FSPL（自由空间路径损耗） −72 dB (40 m, 比 20 m 远一倍多, 损耗加大) ③ 天线增益合计 +3 dBi RSSI = 15 − 72 + 3 = −54 dBm ── 本课：算 SNR ── ④ 噪声底（20 MHz）= −101 dBm SNR = −54 − (−101) = +47 dB ── 判定 MCS ── SNR 47 dB > MCS11 需求 36 dB → 可用 MCS11（1024-QAM） 结论：40 m 仍能跑最高速（5 GHz 短距离、无遮挡时）

再算一个远距离 + 干扰的场景，感受 SNR 怎么掉下来：

弱信号场景（5 GHz, 30 m, 隔一堵墙, 20 MHz） ① 发射功率 +15 dBm ② 路径损耗 FSPL −70 dB 墙再吃 12 dB 墙体衰减 −12 dB ③ 天线增益合计 +3 dBi RSSI = 15 − 70 − 12 + 3 = −64 dBm ④ 噪声底 −101 dBm SNR = −64 − (−101) = +37 dB SNR 37 dB：MCS11（36）勉强，MCS9（29）稳，选 MCS9 结论：隔墙后从 MCS11 掉到 MCS9，速率下降但仍可用
对驱动重要 / For Driver Stack

这两道算例演示的就是自适应速率（rate adaptation）每天在做的判断。你做驱动协议栈时，rate control 算法（minstrel_ht、厂商私有算法）实时测 RSSI/SNR，挑能维持 PER（包错误率） < 10% 的最高 MCS。理解这条链：距离/遮挡 → RSSI → SNR → MCS，调试时你就能定位：吞吐掉了，是 RSSI 掉了（信号弱）还是噪声底升了（干扰）—— 这两类问题的驱动层处理完全不同。

带宽权衡：一个反直觉但关键的点

既然带宽越宽噪声底越高（灵敏度越差），为什么 Wi-Fi 还要不断加宽带宽（20→40→80→160 MHz）？

因为带宽同时决定吞吐。香农定理：容量 = 带宽 · log₂(1 + SNR)。带宽 ×2，理论上吞吐也 ×2（SNR 够高时）。所以 Wi-Fi 的策略是：

  • SNR 高时（近距离）：加宽带宽 → 吞吐暴涨（噪声底升高也够用）
  • SNR 低时（远距离）：缩窄带宽 → 噪声底降低 → 保住 SNR → 维持连接
桥梁 / From sub-GHz

你做 sub-GHz LoRa 时一定调过两个独立参数：BW（带宽）和 SF（扩频因子）。注意这俩是不同的东西，别搞混（我自己上一版课件就搞混过，见下方纠错框）。

真正和 Wi-Fi「信道宽度自适应」对口的，是 BW：LoRa 缩 BW → 噪声底降 → 灵敏度涨但速率掉，这正是上面热噪声公式的直接应用。Wi-Fi 也是同样的权衡，只是方向常被设成「尽量宽」。

而 SF 是 LoRa 特有的扩频增益机制（chirp 持续时间 = 2^SF / BW，SF 越大积分越长 → 处理增益越高 → 灵敏度越好）。这部分在 Wi-Fi 里没有对应物 —— Wi-Fi 用 OFDM 子载波 + 编码增益，不用 chirp 扩频。所以 SF 的权衡经验不能直接迁移到 Wi-Fi，只有 BW 的权衡经验能迁移。

纠错记录 / Correction

本课上一版本我曾把 SF 错误描述为「扩带宽」，并在桥梁框里写「SF 降=扩带宽」。这是错的，SF 不是带宽，是扩频因子（chirp 扫频速度）。已在学员纠正后修正。LoRa 灵敏度高的核心原因是 SF 的处理增益，BW 是次要因素。感谢学员指出。

这也是为什么 Wi-Fi 6（802.11ax）引入了更窄的子载波间隔（78.125 kHz，第 2 课提过）—— 在给定带宽里塞更多子载波，等于「在不增加总带宽的前提下获得更细的资源粒度」，部分缓解了带宽/灵敏度的矛盾。

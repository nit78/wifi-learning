OFDM（正交频分复用）的「正交」到底是什么

上一课你问「OFDM 的正交是什么意思」，答了「子载波相位差 90°」。这个回答需要纠正 —— 不是说你想错了，而是它答的是另一个「正交」。这课的目标，就是让你从此再也不会把两种正交搞混，并且建立 OFDM 频域正交的几何直觉。

这课对你尤其关键，因为你做的是 sub-GHz 窄带协议栈（FSK/LoRa 类，单载波或少量载波）。OFDM「几十上百个子载波并行」是和你过往经验最大的断层 —— 也是 WiFi 能比 sub-GHz 快两个数量级的根本原因。

先纠正：两个「正交」是不同层面

本课支点 / Core Insight

无线电里有两层「正交」，必须分清：

  • I/Q（同相 / 正交）正交（QAM (正交幅度调制) 用）：单个载波内，cos 与 sin 两路，相位差 90°，承载 I、Q 两路基带数据。
  • OFDM 正交（WiFi 用）：多个子载波之间，靠频率间隔精确 = 1/T，频域 sinc（sinc 函数）零点对齐，频谱可重叠而互不干扰。

记口诀：「相位差 90° 是一个波里两路；频率差 1/T 是多个波之间。」

为什么这两层会搞混？因为它们都用「正交」这个词，但指的是不同的数学对象：

关键：WiFi 两个正交都用 —— OFDM 把信道切成多个子载波，每个子载波内部再用 I/Q + QAM。先搞清这层嵌套，后面 MCS（调制与编码方案）、调制阶数才不会乱。

为什么需要 OFDM：sub-GHz 的直觉先放一边

先做个思想实验，唤起你的 sub-GHz 直觉。LoRa / FSK 里，一个载波承载一个数据流。要传更快？两条路：

  1. 加大带宽：FSK 调频更快，但单载波太宽会被多径信道毁掉（不同频率分量到达时间不同 → 信号畸变）。
  2. 用多个载波：把数据分到 N 个窄载波上并行发。问题来了 —— 窄载波之间怎么隔开？

最朴素的办法：每个载波之间留保护频带（guard band），互不重叠 —— 这叫 FDM（Frequency Division Multiplexing，传统频分复用，无线电早期就这么干）。FDM 简单但浪费频谱：保护频带占了大头。

OFDM 的天才之处：能不能让子载波重叠，又互不干扰？答案就是「频率间隔 = 1/T」。这把 FDM 的保护频带全节省了，频谱效率直接翻几倍。

桥梁 / From sub-GHz

你做 sub-GHz 单载波 FSK/LoRa，频谱是一个宽 sinc 主瓣。OFDM 换了个思路：把很多个窄 sinc 并排，让它们的零点对齐到彼此的中心。窄子载波抗多径，密集排布省频谱，两全其美。

核心公式：Δf = 1/T

整节课其实就靠这一条公式撑起来。把它刻进脑子：

OFDM 正交的根条件 / Root condition Δf = 1 / T_symbol

含义：相邻两个子载波的频率差，必须精确等于一个 OFDM 符号周期的倒数。代入 802.11a 的实际数字感受一下：

802.11a 的实际取值 / 802.11a numbers T_symbol = 3.2 μs
Δf = 1 / 3.2 μs = 312.5 kHz
20 MHz 信道 ÷ 312.5 kHz ≈ 64 个子载波（其中 52 个有效，其余做保护/直流）

这个 312.5 kHz 不是随便选的 —— 它必须等于 1/T，否则正交破坏。这也是为什么 802.11 标准里符号周期和子载波间隔是捆绑定义的，不能单独改。

对固件重要 / For Driver Stack

在驱动/协议栈层，你看到的不是「子载波」，而是整条 OFDM 符号。但理解 Δf = 1/T 能解释几个你迟早会遇到的驱动现象：① 802.11ax 把 Δf 从 312.5 kHz 砍到 78.125 kHz（符号周期 ×4），是为了更长符号抗多径，代价是相同带宽下子载波数 ×4；② Doppler（多普勒）频偏（移动场景）会破坏正交 → ICI（子载波间干扰），驱动层会看到 MCS 自动降档。

几何直觉：sinc 零点对齐

公式之外，给你一个能在脑子里画出来的直觉。一个持续 T 秒的纯正弦波（一个子载波），它的频谱长这样（sinc 函数）：

幅度 ↑ │ ● ← 子载波 1 的中心，幅度最高 │ ╱│╲ │ ╱ │ ╲ ● ← 子载波 2 的中心（间距 Δf） │╱ │ ╲ ╱│╲ ──┼───┼───╳───┼──┼──→ 频率 零点 零点 零点 ← 子载波 1 的零点恰好落在子载波 2 的中心 ↑ 子载波 1 的频谱在子载波 2 中心处 = 0

关键：每个子载波频谱的零点（sinc 函数过零的位置），间距正好是 1/T = Δf。所以当你把第二个子载波放在 Δf 处时，它落在第一个子载波的零点上。在接收端，对准某子载波中心采样，其他所有子载波在那里贡献都恰好为零 —— 这就是「正交」。

数学上更精确的说法：任意两个子载波在一个符号周期 T 内做内积（积分），结果为零。内积为零 = 正交 = 互不干扰。WiFi 接收机用一个 FFT（快速傅里叶变换）就能把所有子载波同时分离出来 —— 这也是 OFDM 能用数字电路高效实现的关键（不用几百个模拟滤波器）。

术语：「内积为零」是「正交」的严格定义 —— 几何上就是两个向量互相垂直。OFDM 的子载波在函数空间里互相垂直，所以叫正交。

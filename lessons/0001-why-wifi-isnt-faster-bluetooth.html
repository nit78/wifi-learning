为什么 Wi-Fi 不是「更快的蓝牙」

你已经写过蓝牙固件，会熟练地说出 connection event (连接事件)、piconet (微微网)、FHSS (跳频扩频)。这一课不教你 Wi-Fi 是什么，而是回答一个能让整本 CWNA 突然变薄的问题：

Wi-Fi 和蓝牙一样都跑在 2.4 GHz，为什么 Wi-Fi 能做到几百 Mbps、覆盖几十米，而蓝牙（尤其 BLE）天生就只能跑 1–2 Mbps、十米内？

答案不是「Wi-Fi 用了更好的芯片」。答案是一个架构层面的根本选择 —— 两种技术对「谁来决定什么时候说话」给出了截然相反的回答。理解这个选择，CWNA 后面所有章节（PHY (物理层)、MAC (介质访问控制层)、介质访问、BSS (基本服务集)）都会挂到这一根钉子上。

一句话答案

本课支点 / Core Insight

蓝牙是「中央调度」模型 —— 一个 master (主设备) 决定每个 slot 谁说话；Wi-Fi 是「分布式竞争」模型 —— 每个设备自己抢介质，靠规矩避免打架。

这一个差别，连带决定了 PHY（怎么调制）、拓扑（谁和谁怎么连）、QoS (服务质量)（怎么保证延迟）几乎一切。CWNA 整本书都在讲「分布式竞争」的后果。

术语：调度式 = scheduled / centralized；竞争式 = contention-based / distributed。这两个词在 802.11 文档里到处都是。

两条路线，一张对照表

下面这张表是本课也是整个工作区最常回看的一张表。它会随着学习不断变长（之后还会加 PHY、加密、省电、漫游等行）。先把介质访问这一行的范式差刻进脑子。

注意最后一行 —— 这才是关键。每条路线都付了代价。Wi-Fi 不是「更好」，它是把资源押在了「宽信道 + 高速率」上，代价就是必须解决「很多人同时抢一个宽信道」的问题。CWNA 里你接下来要学的 backoff (退避)、CW (Contention Window, 竞争窗口)、hidden node (隐藏节点 / 隐藏终端)、RTS/CTS (请求发送 / 允许发送)，全都是「分布式竞争」这条路线要还的债。

把蓝牙的脑子翻过来：调度 → 竞争

回忆 BLE 的 connection event：central 在每个 anchor point 醒来，peripheral 在 central 之后发，时刻表是定的。这就是 TDMA (时分多址) —— Time Division，时间被切成格子分配。master/central 是仲裁者，不会有两个设备同时说话，所以不需要「碰撞避免」机制。

桥梁 / From BT

你写过 BLE controller 固件，一定记得那个 anchor point 的精确定时 —— 这是 TDMA 的灵魂：所有设备对齐一个时钟，由一个调度者分发时间。

Wi-Fi 把这个脑子翻过来：没有人发时刻表。一个 AP (接入点) 周围可能有几十个 STA (站点)，谁想发就发 —— 但发之前先听（Carrier Sense），如果信道忙就等一个随机时间再试（Backoff）。这就是 CSMA/CA (载波侦听多址接入 / 碰撞避免) —— Carrier Sense Multiple Access with Collision Avoidance。

「先听后说」听起来朴素，但它是整个 802.11 MAC 子层的心脏。CWNA 第 8、9 章（MAC + 媒质访问）几乎都在讲这个机制的细节与边角情况。

对固件重要 / For Firmware

在 BT controller 固件里，定时精度是核心 KPI（slot 边界、sleep/wake 时序）。到了 Wi-Fi 固件，KPI 变成了 退避计时器的实现、CCA (Clear Channel Assessment, 信道空闲评估) 的灵敏度、ACK/SIFS (短帧间间隔) 的响应时序。换的是从「准时」到「抢得巧 + 反应快」。这一点你之后调 mac80211 / 厂商驱动会反复体会到。

为什么这个选择决定了 PHY：FHSS → OFDM (正交频分复用)

调度式模型里，每个 slot 给一个设备用，瞬时带宽不需要太大 —— 所以蓝牙用 GFSK（一种单载波、低阶的调制），再配上 FHSS 在 79 个信道间跳，用「躲」来避开干扰。窄带 + 跳频，省电、抗窄带干扰，但峰值速率天花板低。

竞争式模型要的是把一个宽信道吃满：谁抢到介质，就尽可能多地塞数据进去，因为下一秒可能就被别人抢走了。这就需要把一个 20 MHz 的信道切成几十到上千个正交子载波，并行传 —— 这就是 OFDM。OFDM 是 CSMA/CA 的天然搭档：宽信道 + 高吞吐，代价是耗电和实现复杂。

记住这条因果链

分布式竞争 → 必须最大化单次抢占的价值 → 必须用宽信道 → 必须把宽带信号拆成正交子载波 → OFDM。

反过来看蓝牙：调度式 → 单 slot 带宽够用 → 窄带 GFSK → 用跳频躲干扰 → FHSS。

这两条路线不是任意选的，是同一个根选择的两个必然结果。这一条想通了，CWNA 第 2 章（RF 基础）、第 6 章（PHY）、第 8 章（MAC）就不再是 6 个孤立章节，而是一根线串起来的。

这课给整本书的钉子

学完这课，CWNA 的目录在你眼里应该变成一根线：

  1. RF 基础 / RF 数学 —— 理解 OFDM 宽带信号怎么传播、衰减、被噪声淹没。
  2. 802.11 标准族（a/b/g/n/ac/ax） —— 看 PHY 怎么从单载波 DSSS 演化到 OFDM、再到 OFDMA。
  3. MAC + 媒质访问 —— CSMA/CA 的全部细节，backoff、CW、SIFS、ACK、RTS/CTS。
  4. PHY —— OFDM 子载波、preamble、调制编码（MCS）。
  5. 拓扑 / 部署 / 勘测 / 安全 —— 作为背景快速过，对固件不是核心。

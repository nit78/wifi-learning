dB 与 dBm —— 从你见过的数字说起

这课从零讲 RF 的数学语言。你做协议栈时见过这些数字（发射功率、RSSI (接收信号强度指示)、灵敏度），但没串成体系、也没自己算过链路预算。今天的目标：让你见到 dB/dBm 不再犯怵，并能亲手算一道完整的链路预算。

我们不走「先讲公式再讲例子」的路子 —— 反过来，先从一个你大概率见过的数字说起。

从一个你见过的数字开始：RSSI −65 dBm

你调 sub-GHz 或 BT 参数时，大概率在日志或工具里见过类似 −65 dBm 这样的 RSSI 值。这个数字告诉你接收到的信号有多强。但「−65 dBm」到底是什么意思？为什么是负数？为什么用 dBm 而不说「0.0000000316 瓦」？

这三个问题，答案都指向同一件事：RF 世界的功率值跨度太大，用「瓦」根本没法读。

本课支点 / Core Insight

dB 和 dBm 是 RF 工程师发明的「速记法」，为了对付跨 12 个数量级的功率范围。它们不是高深的物理概念，就是一个对数换算，让 −100 到 +30 这种小范围数字代替 0.0000000000001 到 10 这种让人读不下去的瓦数。

为什么 RF 要用 dB：一个动机实验

假设你不用 dB，全用瓦（W）或毫瓦（mW）来描述功率。看看 Wi-Fi 实际场景里的功率跨度：

场景功率（mW）写成小数读起来怎样
AP 发射功率 100 mW 0.1 W 还好
手机 BLE 发射 1 mW 0.001 W 还行
近距离接收（强） 0.001 mW 0.000001 W开始难读
中等距离接收 0.00001 mW10⁻⁸ W 数零数到眼花
灵敏度极限（弱） 0.0000001 mW10⁻¹⁰ W没法读

看到了吗？从 AP 发射到接收灵敏度，功率跨越 9 个数量级（100 mW 到 10⁻¹⁰ W）。用线性瓦数写，全是零，根本没法心算比较。

对数（log）就是为这种场景生的：把「乘除」变成「加减」，把「数量级」变成「一位数」。log₁₀(100) = 2，log₁₀(0.0001) = −4。一个 −4 比四个零好读太多。这就是 dB 存在的全部动机。

dB：两个功率比值的对数

先把dB 讲清楚。dB 是比值，不是绝对量：

dB 的定义 / Definition of dB dB = 10 · log₁₀(P₁ / P₂)

意思是：拿两个功率相除，再取 log₁₀，再 ×10。结果就是一个「倍数」的对数表示。比如：

  • 功率 ×2：10·log₁₀(2) = 10 × 0.3 ≈ +3 dB
  • 功率 ×10：10·log₁₀(10) = +10 dB
  • 功率 ×100：10·log₁₀(100) = +20 dB
  • 功率 ×1000：10·log₁₀(1000) = +30 dB
  • 功率 减半 (×0.5)：10·log₁₀(0.5) ≈ −3 dB
  • 功率 ÷10：−10 dB
为什么是 ×10？

因为原始 Bel 单位（log₁₀ 比值）单位太大，乘 10 让它更细。1 Bel = 10 dB。Bel 因 Bell 实验室得名，dB = decibel（deci = 十分之一）。

你不需要每次都算 log。背熟下面这 6 个对应，就能读 90% 的 RF 文档：

功率倍数dB怎么记
×1（不变） 0 dB log₁₀(1) = 0
×2 +3 dB log₁₀(2) ≈ 0.3
×4 +6 dB = 2 × (×2) = 3+3
×10 +10 dBlog₁₀(10) = 1
×100 +20 dB= 10 + 10
×1000 +30 dB= 10 + 10 + 10
关键洞察：dB 是加减法。×4 不是 ×2 的两倍 dB，是 +6 dB（= +3 + 3）。因为 dB 是对数，对数里乘法变加法。这就是 dB 的全部魔力。

dBm：把「比值」锚定到 1 mW

dB 是比值，但工程师需要说「这个 AP 发射功率具体是多少」。于是把比值的分母固定为 1 mW，得到绝对单位 dBm：

dBm 的定义 / Definition of dBm P(dBm) = 10 · log₁₀( P(mW) / 1 mW ) = 10 · log₁₀( P(mW) )

所以 0 dBm = 1 mW。其它值都从这推：

dBmmW怎么来的典型场景
0 dBm 1 mW 定义基准 BLE 典型发射
+10 dBm 10 mW ×10 = +10 dB 低功耗 IoT Wi-Fi
+20 dBm 100 mW ×100 = +20 dB 典型 Wi-Fi AP / 手机最大
+30 dBm 1 W ×1000 = +30 dB 5 GHz 室外 AP 上限
−10 dBm 0.1 mW ÷10 = −10 dB 近距离强信号
−60 dBm 0.000001 mW÷10⁶ 极好信号
−80 dBm 10 pW ÷10⁸ 中等可用
−100 dBm 0.1 pW ÷10¹⁰ 近灵敏度极限
为什么 RSSI 是负数？

因为接收功率永远小于 1 mW（除非你贴着 AP）。0 dBm = 1 mW 是基准，接收功率比 1 mW 小 → 取 log 是负数 → dBm 是负数。RSSI −65 dBm 意思是「比 1 mW 弱 65 dB」，即 1 mW ÷ 10⁶·⁵ ≈ 3.16 × 10⁻⁷ mW。负号不是坏事，只是因为接收功率天生很小。

实战读数：−30 ~ −40 dBm = 极强（贴着 AP），−65 dBm = 好，−75 dBm = 可用，−85 dBm = 边缘，−95 dBm 以下 = 灵敏度极限。这五个档位记住，看 RSSI 日志就有感觉了。

dBm 之间的运算：全是对数加减

这是 RF 工程师每天用的核心技能。dBm 之间相加，等于功率相乘；dBm 加上 dB，等于功率乘以倍数。链路预算全靠这条。

算例 1：功率翻倍 20 dBm = 100 mW 功率翻倍 → +3 dB 20 dBm + 3 dB = 23 dBm = 200 mW
算例 2：信号衰减 发射 20 dBm，路径损耗 60 dB 接收 = 20 dBm − 60 dB = −40 dBm 含义：信号从 100 mW 衰减到 0.0001 mW（÷10⁶）
算例 3：天线增益 发射 15 dBm，天线增益 5 dBi 有效辐射 = 15 dBm + 5 dB = 20 dBm EIRP (等效全向辐射功率) 含义：定向方向上等效于 100 mW 全向发射

看出规律了吗？所有 RF 计算最终都是 dB 域的加减法。发射功率、损耗、增益、灵敏度，全用 dB/dBm 表示，然后像账本一样加减。这就是「链路预算（link budget）」这个名字的由来 —— 它就是一本功率账。

链路预算：一道加减法，预测能不能通

把上面所有东西串起来，就是链路预算。它回答一个问题：「给定发射功率、距离、频率，接收端信号够不够强？」

我们做一道完整的，每一步都解释。场景：手机连 5 GHz Wi-Fi AP，距离 20 米。

完整链路预算 / Full Link Budget（手机 → AP, 5 GHz, 20 m） ① 发射功率（手机） Tx Power = +15 dBm 手机典型最大 ② 发射天线增益 Tx Ant Gain = +0 dBi 手机天线几乎无增益 ③ 自由空间路径损耗 FSPL = −66 dB 下方解释怎么来的 ④ 接收天线增益（AP） Rx Ant Gain = +3 dBi AP 天线稍好 ⑤ 接收馈线损耗 Rx Loss = −1 dB AP 内部走线 ───────────────────────────────────────────────────── 接收信号功率 RSSI = +15 + 0 − 66 + 3 − 1 = −49 dBm （对照实战读数：−49 dBm = 很强的信号，贴 AP 级别）

③ 那个 FSPL = −66 dB 怎么来的？用自由空间路径损耗公式：

自由空间路径损耗 / Free Space Path Loss FSPL(dB) = 20·log₁₀(d) + 20·log₁₀(f) + 32.44
（d = 距离 km，f = 频率 MHz）
关于 66 dB 这个数（教学坦白）

代入 5 GHz、20 m（=0.02 km），完整公式算出约 66 dB。这个数对 5 GHz 室内 20 米是合理量级。要点不是手算精确值，而是掌握方法：所有 RF 损耗都是 dB 加减。实战中你会用工具或厂商 SDK 算精确值，但脑子里能估算量级是基本功。别死记数字，记方法。

有了 RSSI，怎么知道够不够？对照灵敏度

算出 RSSI = −49 dBm 后，怎么判断「能不能通」？对照接收机的灵敏度。灵敏度是接收机在指定包错误率（PER，通常 10%）下能解调的最低信号功率。

典型 Wi-Fi 灵敏度（简化）：

MCS (调制与编码方案) 档调制灵敏度（约）含义
MCS0 BPSK (二进制相移键控) 1/2 −93 dBm 最低速率，最弱信号也能收
MCS3 16-QAM (正交幅度调制) 1/2 −78 dBm 中速
MCS7 64-QAM 3/4 −64 dBm 高速，需较强信号
MCS9 256-QAM 5/6−47 dBm 最高速，需极强信号

回到我们的 RSSI = −49 dBm：它比 MCS9 灵敏度（−47 dBm）还低 2 dB → MCS9 不稳定。但比 MCS7（−64 dBm）高 15 dB → MCS7 稳稳的。所以驱动会选 MCS7 或 MCS8。

这就是自适应速率（rate adaptation）的物理依据 —— 实时测 RSSI/SNR (信噪比)，挑能达到 PER < 10% 的最高 MCS。你做协议栈迟早会调这块逻辑。

三道题，做完就能确认 dB/dBm 是否真的上手了。算不出来也没关系 —— 回看算例再来。

这课的钉子，和下一课

学完这课，你应该能做到：

  • 见到 +3 dB 知道是 ×2，+10 dB 是 ×10，−20 dB 是 ÷100
  • 见到 20 dBm 知道是 100 mW，−80 dBm 知道是「中等可用信号」
  • 能用 dB 加减算一道完整链路预算，得到 RSSI
  • 用 RSSI 对照灵敏度表，判断可达 MCS
  • 明白为什么 RSSI 是负数（接收功率天生小于 1 mW）

没掌握热噪声和 SNR 公式没关系 —— 那是下一课（第 4 课）的内容，会从「为什么 Wi-Fi 灵敏度比蓝牙差」切入，讲噪声底和 SNR。本课只要求 dB/dBm 和链路预算扎实。
